# 인문잡

역사학·철학·미술사학·문헌정보학 분야의 교수, 강사, 연구원 및 박사후연구원 채용공고를 선별해 보여주는 모바일·PC 겸용 웹앱입니다.

## 제공 기능

- 분야별·직종별 공고 검색과 마감 임박순 정렬
- 학교·학과·모집 인원·마감일·지원 방법 표준화
- 여러 학교 관심 등록 및 현재 기기에 저장
- 관심 학교 신규 공고·마감·변경 알림 설정
- 모바일 홈 화면에 설치할 수 있는 PWA
- 매일 오전 7시(한국시간) 공개 공고 자동 갱신
- 대학·기관 공식 웹페이지와 PDF를 찾아 학과·인원·지원방법·자격·제출서류 보강
- 일반 요청으로 열리지 않는 외부 페이지는 브라우저 엔진으로 한 번 더 확인
- 수집에 실패하면 마지막 정상 데이터를 보존

초기 화면에 표시되는 한빛대학교 등의 공고는 화면 확인용 예시이며 실제 공고가 아닙니다. 자동 수집이 처음 성공하면 `data/jobs.json`의 실제 공고로 교체됩니다. 현재 수집기는 하이브레인넷의 역사학·철학 전공 검색과 미술사·문헌정보 관련 키워드 검색 목록을 사용합니다.

## GitHub Pages 배포

1. 이 폴더의 **내용 전체**를 새 GitHub 저장소의 최상위에 올립니다.
2. 기본 브랜치 이름을 `main`으로 사용합니다.
3. 저장소의 **Settings → Pages → Build and deployment → Source**에서 **GitHub Actions**를 선택합니다.
4. **Actions** 탭에서 `Daily recruitment update and Pages deploy`를 한 번 수동 실행합니다.
5. 작업이 끝나면 Pages 주소에서 사이트를 확인합니다.

자동 작업은 매일 한국시간 오전 7시에 실행되며, `data/jobs.json`이 변경되면 봇이 새 데이터를 저장소에 커밋합니다. 저장소의 Action 권한이 읽기 전용이라면 **Settings → Actions → General → Workflow permissions**에서 읽기 및 쓰기 권한을 허용해야 합니다.

### 공식 외부 공고 검색 정확도 높이기

기본 상태에서도 공개 웹 검색 결과를 사용하지만, 네이버 검색 Open API를 연결하면 결과가 더 안정적입니다. 네이버 개발자센터에서 검색 API 애플리케이션을 만든 뒤 GitHub 저장소의 **Settings → Secrets and variables → Actions**에 다음 Repository secret을 등록하세요.

- `NAVER_CLIENT_ID`
- `NAVER_CLIENT_SECRET`

비밀값은 HTML이나 데이터 파일에 기록되지 않고 GitHub Action 실행 중에만 사용됩니다. 등록하지 않아도 작업은 실행되지만 공개 검색 화면 구조가 변경되면 외부 원문 탐색 성공률이 낮아질 수 있습니다.

## Windows에서 로컬로 보기

프로젝트 폴더의 `로컬에서보기.bat`를 더블클릭하면 기본 브라우저에서 앱이 열립니다. 함께 열린 안내 창을 닫거나 Enter 키를 누르면 로컬 실행도 종료됩니다.

`index.html`을 직접 더블클릭하는 방식은 브라우저 보안 정책 때문에 `data/jobs.json`을 불러오지 못할 수 있습니다. 로컬 실행 파일 또는 GitHub Pages 주소를 사용하세요. 로컬 실행에는 Python 3이 필요합니다.

## 알림 기능의 범위

관심 학교와 알림 선택값은 IP 주소가 아니라 브라우저의 기기 저장소에 보관됩니다. IP는 변동·공유될 수 있어 사용자를 안정적으로 구별할 수 없으며, 개인정보 보호 측면에서도 관심 정보 연결 키로 사용하지 않는 것이 좋습니다.

현재 GitHub Pages 버전에는 다음이 구현되어 있습니다.

- 여러 관심 학교 등록
- 알림 종류 선택
- 브라우저 알림 권한 요청
- 서비스 워커의 푸시 알림 수신 준비

사이트가 닫힌 상태에서도 실제 모바일 푸시를 보내려면 Firebase Cloud Messaging, OneSignal 또는 별도 Web Push API가 추가로 필요합니다. 정적 GitHub Pages는 알림 구독 정보를 안전하게 저장할 서버가 없기 때문입니다. 연결 시에는 `sw.js`의 `push` 처리기를 그대로 활용할 수 있습니다.

## 데이터 수집 시 주의사항

- 배포 전에 하이브레인넷 및 연결된 대학 사이트의 이용약관, robots.txt와 재사용 조건을 확인하세요.
- 수집 빈도를 과도하게 높이지 마세요.
- 원문 전체를 복제하기보다 채용 사실정보와 공식 링크만 제공하세요.
- 수집된 정보는 자동 해석 결과이므로 관리자가 정기적으로 인원·마감일·지원 주소를 검수해야 합니다.
- 사이트 구조가 변경되면 `scripts/update_jobs.py`의 목록 및 상세 추출 규칙을 수정해야 합니다.
- 하이브레인넷 상세 페이지는 자동 요청을 차단할 수 있습니다. 이 경우에도 목록에 공개된 제목·접수기간·등록일·원문 주소는 수집되며, 학과·모집 인원·지원 방법은 `원문 확인 필요`로 표시됩니다.
- 상세 페이지가 차단되면 공고 제목을 기준으로 대학·기관 공식 도메인을 검색하고, 공식 게시판이나 PDF에서 내용을 보강합니다. 공식 도메인이 아니거나 제목 유사도가 낮은 결과는 채택하지 않습니다.
- 외부 원문 결과는 자동 해석 정보이므로 `외부 원문 자동 수집`으로 표시되며 최종 지원 전 공식 페이지 확인이 필요합니다.

## 파일 구성

```text
index.html                         화면과 사용자 기능
data/jobs.json                    화면에 표시할 채용 데이터
scripts/update_jobs.py            공개 공고 수집·분류 도구
scripts/enrich_jobs.py            대학 공식 원문·PDF 탐색 및 상세정보 보강
.github/workflows/                매일 갱신 및 Pages 배포 자동 작업
manifest.webmanifest, sw.js       설치형 웹앱과 알림 기반
LICENSE.md                        사용·복제 제한 조건
```
